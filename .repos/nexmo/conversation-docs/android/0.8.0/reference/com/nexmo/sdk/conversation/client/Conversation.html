Class Overview

A container that you use to manage communications between Members, the conversation history and context.

Users cannot manually create conversation objects, but creating a Conversation via newConversation(String, ConversationCreateListener) will dispatch a new Conversation object.

Every action on a conversation will dispatch a completion callback that contains an updated Conversation object. For ex: 

     myConversation.sendText("payload", new EventSendListener() {
         @Override
         public void onSent(Text text) {
              // Update the application UI here if needed with updated conversation.
         }
         @Override
         public void onError(int errCode, String errMessage) {
              // Update the application UI here if needed.
         }
     });

Listen for incoming text and image events addEventListener(EventListener)

Example usage:

     myConversation.addEventListener(new EventListener() {
         @Override
         public void onTextReceived(Event event) {
              // Update the application UI here if needed.
         }
         @Override
         public void onTextDeleted(Event event) {
              // Update the application UI here if needed.
         }
         @Override
         public void onImageReceived(Event event) {
              // Update the application UI here if needed.
         }
         @Override
         public void onImageDeleted(Event event) {
              // Update the application UI here if needed.
         }
         @Override
         public void onImageDownloaded(Image image) {
              // Update the application UI here if needed.
         }
     });

Sending a text message to a specific conversation is achieved using sendText(String, EventSendListener)

Example usage:

     myConversation.sendText("message payload...", new EventSendListener() {
         @Override
         public void onSent(Event event) {
              // Update the application UI here if needed.
         }
         @Override
         public void onError(int errorCode, String message) {
              // Update the application UI here if needed.
         }
    });

Listen for typing indicator events addTypingListener(MemberTypingListener)

Example usage:

     myConversation.addTypingListener(new MemberTypingListener() {
         @Override
         public void onTyping(Member member, Member.TYPING_INDICATOR typingIndicator);
              // Update the application UI to highlight when someone else is typing.
         }
     });

public ImageUploader.CancelableCall sendImage (String imagePath, EventSendListener imageSendListener)

experimental feature

Send/Upload an Image event to a conversation.

Once the image is uploaded, you will be able to access all 3 representations:

  • Original at 100% size via getOriginal()
  • Medium at 50% size via getMedium()
  • Thumbnail at 10% size via getThumbnail()

Each ImageRepresentation contains a ImageRepresentation#bitmap that can be used to update UI

For listening to incoming/sent events events, register using addEventListener(EventListener)

Note: at this only the image path is allowed, but bitmap or file will soon be added.

Parameters
imagePath The image location, mandatory.
imageSendListener The completion listener, mandatory.
Returns
  • object to control upload process or null in case of error

public void update (ConversationListener conversationListener)

Retrieve full conversation information, like members and conversation details. available information includes:

  • getDisplayName()
  • getCreationDate()
  • getMembers()

For retrieving the text/image events use updateEvents(String, String, ConversationListener) (String, String, ConversationListener)}.

Please bear in mind to refresh the members list on a regular basis to avoid events from 'unknown' members.

For the first time launching the app, the local cache will not present any results.

Parameters
conversationListener The listener in charge of dispatching the result.
